About HTML

HTML (HyperText Markup Language) is the standard markup language used to create and structure content on the web. It uses tags to describe elements such as headings, paragraphs, links, images and more. HTML defines the skeleton of a webpage — CSS styles it and JavaScript makes it interactive.

Core Concepts

  • Elements and Tags: <tag>content</tag>
  • Attributes: provide extra information — e.g., src, alt, href
  • Document tree: DOM (Document Object Model)
  • Semantic HTML: use meaningful tags like <header>, <nav>, <article>

Basic Page Example

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>Simple Page</title>
  </head>
  <body>
    <h1>Hello HTML</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

History of HTML

HTML was created by Tim Berners-Lee in 1991 to share scientific documents. Over time, HTML developed through multiple versions (HTML 2.0, 3.2, 4.01) and XHTML. HTML5 (released as a W3C Recommendation in 2014) modernized the language with new semantic elements, multimedia support and APIs.

YearMilestone
1991Original HTML proposal by Tim Berners-Lee
1995–1997HTML 2.0 and 3.2 — early standards
1999HTML 4.01 — focus on separation of concerns
2008–2014HTML5 development and recommendation

Common HTML Tags & Their Formats

Below are frequently used tags with short descriptions and code examples.

Structure & Text

  • <!DOCTYPE html> — declares HTML5 document type.
  • <html> — root element.
  • <head> — meta information (title, links).
  • <body> — page content.
  • <h1>…<h6> — headings (h1 is largest).
  • <p> — paragraph.
  • <a href="url"> — anchor/link.

Media & Layout

  • <img src="..." alt="..."> — image (self-closing in HTML5).
  • <video> — video; supports <source>.
  • <section>, <article>, <aside> — semantic blocks.
  • <div> — generic container.
  • <nav> — navigation links.

Lists & Inline

  • <ul> <li> — unordered list.
  • <ol> <li> — ordered list.
  • <span> — inline generic container.
  • <strong>, <em> — semantic emphasis.

Code & Preformatted

<pre>Preformatted text</pre>
<code>Inline code</code>
<blockquote>A quote</blockquote>

Tables — How to Create & Describe

Tables allow you to show tabular data. Use <table> with <thead>, <tbody>, <tr>, <th> and <td> for structure and accessibility.

Basic Table Markup

<table>
  <thead>
    <tr><th>Name</th><th>Age</th></tr>
  </thead>
  <tbody>
    <tr><td>Alice</td><td>24</td></tr>
  </tbody>
</table>

Rendered Example

NameRoleCountry
AliceDeveloperIndia
BobDesignerUSA
ClaraManagerUK

Accessibility Tips

  • Use <caption> for tables that require a title.
  • Use <th scope="col"> and <th scope="row"> for screen readers.
  • Keep tables responsive — consider horizontal scrolling on small screens.

Forms — Inputs, Checkboxes & Everything

Forms gather user input. Each input should have a label. Use proper types and attributes (required, minlength, pattern) for better UX and validation.

Form Markup (Code)

<form action="/submit" method="post">
  <label>Name<input type="text" name="name" required></label>
  <label>Email<input type="email" name="email" required></label>

  <fieldset>
    <legend>Choose your interests</legend>
    <label><input type="checkbox" name="interest" value="design"> Design</label>
    <label><input type="checkbox" name="interest" value="dev"> Development</label>
  </fieldset>

  <label>Message<textarea name="msg" rows="4"></textarea></label>
  <button type="submit">Send</button>
</form>

Form Tips & Best Practices

  • Always pair inputs with <label for="id"> for accessibility.
  • Use appropriate type attributes: email, tel, number.
  • Prefer server-side validation; use client-side validation for better UX.
  • Group related inputs with <fieldset> and <legend>.
